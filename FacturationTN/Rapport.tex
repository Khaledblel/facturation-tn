\section{Introduction}
FacturationTN est une application Blazor Web App (.NET 10) pour gerer clients, produits et factures avec conformite fiscale tunisienne (TVA multi-taux et timbre fiscal).

\subsection{Objectifs}
\begin{itemize}
\item Gestion complete des clients, produits, factures
\item Calcul automatique HT / TVA / TTC et timbre fiscal
\item Authentification utilisateur et pages protegees
\item Tableau de bord avec indicateurs et graphiques
\end{itemize}

\section{Architecture}
\begin{itemize}
\item \textbf{Frontend}: Blazor Components (Razor pages + layouts)
\item \textbf{Backend}: Services metier injectes via DI
\item \textbf{Data}: EF Core + SQLite (Code First + migrations)
\item \textbf{Security}: Cookie Authentication + Authorization
\end{itemize}

\shot{screenshots/architecture.png}{14cm}{Architecture de l'application}

\section{Modele de donnees}
Entites principales: \texttt{Client}, \texttt{Produit}, \texttt{Facture}, \texttt{LigneFacture}, \texttt{UserAccount}.

\subsection{Regles metier facture}
\begin{itemize}
\item Numero auto: FA-YYYY-NNNN
\item Workflow: Brouillon $\rightarrow$ Validee $\rightarrow$ Payee / Annulee
\item Total HT = somme des lignes HT
\item Total TVA = somme des lignes TVA
\item Total TTC = HT + TVA + Timbre
\end{itemize}

\section{Alignement pedagogique (TP1-TP10)}
Le projet suit les principes et la logique des TP etudies:
\begin{itemize}

\item Utilisation EF Core + migrations + LINQ async (TP5/TP6)
\item Formulaires \texttt{EditForm} et validation (TP7)
